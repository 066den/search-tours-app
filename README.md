# Search Tours App

Клієнтська частина сервісу пошуку турів. Тестове завдання на розробку React-застосунку для пошуку та перегляду турів.

## Технологічний стек

- **Vite** - збірщик та dev-сервер
- **React 19** - UI бібліотека
- **TypeScript** - типізація
- **React Router DOM** - маршрутизація
- **Zustand** - управління станом
- **Tailwind CSS v4** - стилізація
- **ESLint** - лінтінг коду
- **Prettier** - форматування коду

## Структура проєкту

```
src/
├── components/     # Перевикористовувані компоненти
├── hooks/          # Кастомні хуки
├── pages/          # Сторінки застосунку
│   ├── Home.tsx
│   └── TourDetails.tsx
├── store/          # Zustand stores
│   └── useToursStore.ts
├── types/          # TypeScript типи
│   └── index.ts
├── utils/          # Утиліти
├── App.tsx         # Головний компонент з роутингом
└── main.tsx        # Точка входу
```

## Встановлення та запуск

### Вимоги

- Node.js версії 20.19.0 або вище (або 22.12.0+)
- npm або yarn

### Покрокова інструкція з розгортання

1. **Клонуйте репозиторій** (якщо ще не клоновано):

   ```bash
   git clone <repository-url>
   cd search-tours-app
   ```

2. **Встановіть залежності**:

   ```bash
   npm install
   ```

3. **Запустіть застосунок у режимі розробки**:

   ```bash
   npm run dev
   ```

4. **Відкрийте браузер** та перейдіть за адресою:
   ```
   http://localhost:5173
   ```

Застосунок буде автоматично перезавантажуватися при зміні файлів.

### Збірка для продакшену

```bash
npm run build
```

Зібрані файли будуть знаходитися у папці `dist/`

### Перегляд продакшен збірки

```bash
npm run preview
```

### Лінтінг

```bash
npm run lint
```

### Форматування коду

```bash
npm run format
```

Перевірка форматування:

```bash
npm run format:check
```

## Функціональність

Застосунок надає:

- Форму пошуку турів з вибором напрямку та міста відправлення
- Список турів, відсортований за ціною (від меншої до більшої)
- Перегляд повної інформації про тур
- Окрему сторінку для кожного туру

## Розробка

### Правила розробки

- Заборонено використовувати готові бібліотеки компонентів (Ant Design, MUI, Bootstrap тощо)
- Дозволено застосовувати лише низькорівневі утиліти (наприклад, @floating-ui для позиціонування)
- Усі компоненти (input, dropdown, картки, кнопки тощо) повинні бути реалізовані самостійно
- Код повинен відповідати стандартам якості production-коду

### Коміти

Коміти варто робити поетапно — відповідно до виконаних завдань.
