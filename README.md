# Search Tours App

Клиентская часть сервиса поиска туров. Тестовое задание на разработку React-приложения для поиска и просмотра туров.

## Технологический стек

- **Vite** - сборщик и dev-сервер
- **React 19** - UI библиотека
- **TypeScript** - типизация
- **React Router DOM** - маршрутизация
- **Zustand** - управление состоянием
- **Tailwind CSS v4** - стилизация
- **ESLint** - линтинг кода
- **Prettier** - форматирование кода

## Структура проекта

```
src/
├── components/     # Переиспользуемые компоненты
├── pages/          # Страницы приложения
│   ├── Home.tsx
│   └── TourDetails.tsx
├── store/          # Zustand stores
│   └── useToursStore.ts
├── types/          # TypeScript типы
│   └── index.ts
├── utils/          # Утилиты
├── App.tsx         # Главный компонент с роутингом
└── main.tsx        # Точка входа
```

## Установка и запуск

### Требования

- Node.js версии 20.19.0 или выше (или 22.12.0+)
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`

### Сборка для продакшена

```bash
npm run build
```

Собранные файлы будут находиться в папке `dist/`

### Просмотр продакшен сборки

```bash
npm run preview
```

### Линтинг

```bash
npm run lint
```

### Форматирование кода

```bash
npm run format
```

Проверка форматирования:

```bash
npm run format:check
```

## Функциональность

Приложение предоставляет:

- Форму поиска туров с выбором направления и города отправления
- Список туров, отсортированный по цене (от меньшей к большей)
- Просмотр полной информации о туре
- Отдельную страницу для каждого тура

## Разработка

### Правила разработки

- Запрещено использовать готовые библиотеки компонентов (Ant Design, MUI, Bootstrap и т.д.)
- Разрешено использовать только низкоуровневые утилиты (например, @floating-ui для позиционирования)
- Все компоненты (input, dropdown, карточки, кнопки и т.д.) должны быть реализованы самостоятельно
- Код должен соответствовать стандартам качества production-кода

### Коммиты

Коммиты должны быть поэтапными, соответствующими выполненным задачам.
